# Amiga Project

Playground for Amiga-related tests, libraries, and CLI tools targeting AmigaOS 3.2+.

## Overview

This repository contains a collection of C libraries and tools developed for AmigaOS. It includes a core support library (`AppSupport`), various command-line utilities (`Shelltools`), and educational examples (`BuildingBlocks`).

## Stack & Requirements

- **Language:** C (C99 standard).
- **Target OS:** AmigaOS 3.2+.
- **Compilers:**
  - [Bebbo's amiga-gcc](https://franke.ms/git/bebbo/amiga-gcc) (Recommended for cross-compiling on Linux).
  - `vc` (vbcc) or other native Amiga compilers.
- **Dependencies:**
  - AmigaOS 3.2 NDK.
  - [Roadshow](http://roadshow.amiga-projects.net/) (for network features).
  - AmiSSL SDK (for HTTPS support).
  - [SQLite](https://www.sqlite.org/) (required for `sqliteclient`).
  - ttengine-7.2 (optional, used in some components).

## Project Structure

- **`src/AppSupport/`**: Core library with generic structures and helper functions:
    - **Archive**: Basic archive management (listing, extracting, adding files).
    - **Collections**: Data structures (Linked lists, Hash tables/Dictionaries, Binary Trees).
    - **Environment**: Host system information (language, charset, color mode).
    - **Filetools**: Temporary file and directory utilities.
    - **Image**: IFF ILBM image processing and ASCII conversion.
    - **Network**: HTTP/HTTPS fetching (requires Roadshow & AmiSSL).
    - **Shell**: Colored text output for the Amiga shell.
    - **Testutils**: Macros and helpers for unit testing.
    - **Windows**: Window querying and manipulation.
- **`src/Shelltools/`**: Useful CLI utilities:
    - `arch`: Archive management tool (list, add, extract, replace).
    - `cldir`: Colored directory listings.
    - `fetch`: HTTP/HTTPS file downloader.
    - `img2ascii`: Convert images to ASCII art using colored text.
    - `mktemp`: Create temporary files/directories.
    - `shprintf`: Colored shell printf utility.
    - `shsetcolor`: Set shell output color.
    - `shresetcolor`: Reset shell output color.
- **`src/AppSupportTests/`**: Test suite for the `AppSupport` library.
- **`src/BuildingBlocks/`**: Code examples for specific library usage (Graphics, SQLite, Tilemaps).
- **`docs/`**: Additional documentation (e.g., [BUILD.md](docs/BUILD.md)).

## Setup & Build

### Cross-compiling (Linux)

Using CMake and Bebbo's m68k-amigaos-gcc is the preferred method.

1.  **Configure paths**: Edit `CMakeLists.txt` to point to your local installations of SQLite, AmiSSL, and other libraries.
2.  **Build**:
    ```bash
    mkdir build
    cd build
    cmake .. -DCMAKE_TOOLCHAIN_FILE=path/to/amiga-gcc.cmake
    make
    ```

### Native Compiling (Amiga)

A basic `Makefile` is provided for use with `vc` (vbcc).

1.  **Adjust paths**: Open `Makefile` and ensure `NDK_INC` and other paths are correct.
2.  **Run**:
    ```bash
    make all
    ```

## Scripts & Entry Points

Main executables generated by the build system:

- **Shell Tools**: `arch`, `cldir`, `fetch`, `img2ascii`, `mktemp`, `shprintf`, `shsetcolor`, `shresetcolor`.
- **Tests**: `archivetest`, `archivetest2`, `fetchtest`, `shelltest`.
- **Examples**: `gfxfunctions`, `sqliteclient`, `tilemapfunctions`.

## Environment Variables

The `AppSupport/Environment` module respects the following AmigaOS environment variables:

- `shellcolors`: Controls output coloring. Values: `4`, `amiga` (4 colors); `ansi`, `full` (ANSI colors).
- `charset`: Preferred character encoding (defaults to `ISO-8859-1` or system default).
- `language`: Preferred system language (e.g., `english`).
- `languagename`: Descriptive name of the preferred language.

## Tests

Tests are located in `src/AppSupportTests/`. You can run them after building the corresponding executables (e.g., `archivetest`).
